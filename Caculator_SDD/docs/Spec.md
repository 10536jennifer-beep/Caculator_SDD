# Specification (產品規格書)

## 1. 專案概述 (Overview)
本專案旨在開發一個基於 **WinUI 3** 的現代化桌面計算器應用。目標是提供清晰、反應靈敏的計算體驗，支援標準四則運算與基礎科學運算功能。

## 2. 使用者故事 (User Stories)

### US-001: 標準運算
> **As a** 一般使用者  
> **I want** 能進行基本的加、減、乘、除運算  
> **So that** 我可以處理日常的數字計算需求。

### US-002: 科學運算
> **As a** 進階使用者 (學生或工程師)  
> **I want** 能使用三角函數 (Sin, Cos) 和開根號 (Sqrt)  
> **So that** 我不需要切換到實體工程計算機就能完成作業。

### US-003: 錯誤處理 (防呆機制)
> **As a** 使用者  
> **I want** 當我輸入無效算式 (如除以零) 時，系統能顯示友善提示  
> **So that** 應用程式不會直接崩潰 (Crash) 或顯示奇怪的符號。

## 3. 驗收標準 (Acceptance Criteria, AC)

### 針對 US-001 (標準運算)
* [ ] **AC-1.1**: 輸入兩個整數或浮點數進行加減乘除，結果顯示正確 (誤差容許範圍 `1e-8` 內)。
* [ ] **AC-1.2**: 支援連續運算 (例如 `1 + 2 + 3 =` 應顯示 `6`)。
* [ ] **AC-1.3**: 按下 `C` (Clear) 按鈕應清空當前輸入與運算結果，回復為 `0`。

### 針對 US-002 (科學運算)
* [ ] **AC-2.1**: 點擊 `Sin`, `Cos` 時，應針對當前顯示的數字進行計算 (預設使用 **角度制 Degree**)。
* [ ] **AC-2.2**: 點擊 `Sqrt` (√) 時，應計算當前數字的平方根。

### 針對 US-003 (錯誤處理)
* [ ] **AC-3.1**: 當嘗試除以 `0` 時，顯示器應顯示文字 `"Error"` 或 `"無法除以零"`。
* [ ] **AC-3.2**: 當對負數執行開根號 (`Sqrt(-1)`) 時，顯示器應顯示 `"Error"`。
* [ ] **AC-3.3**: 在顯示 Error 狀態下，按下任何數字鍵或清除鍵應能重置計算器狀態。

## 4. 反向目標 (Anti-Goals) - 本版本不做的功能
* ❌ 不支援歷史紀錄 (History) 保存。
* ❌ 不支援工程師模式 (Hex/Bin 運算)。
* ❌ 不支援圖形繪製 (Graphing)。
* ❌ 不支援鍵盤輸入 (僅限滑鼠點擊 UI 按鈕)。
